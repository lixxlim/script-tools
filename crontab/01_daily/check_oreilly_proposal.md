# 변경내용
- 현재 `md5sum`으로 웹페이지 전체 HTML을 해시하는 방식 대신, 페이지에서 책 리스트 영역만 추출해 정규화한 뒤 `sha256sum`으로 비교하는 Bash 스크립트로 바꾼다.
- 스크립트는 Python과 완전히 분리된 독립 실행형 Bash로 유지한다.
- cron에서는 매일 페이지를 확인하고, 목록 해시가 이전 값과 다를 때만 GitHub Actions를 실행한다.

# 변경이유
- 웹페이지 전체를 해시하면 광고, 공백, 레이아웃, 비본질적인 HTML 변경에도 반응할 수 있다.
- 책 리스트 영역만 비교하면 실제 책 추가, 삭제, 변경만 더 안정적으로 감지할 수 있다.
- Bash + `curl` + `sha256sum` 조합은 cron 환경에서 단순하고 유지보수가 쉽다.
- `md5`보다 `sha256`을 쓰면 향후 확장이나 재사용 관점에서 더 무난하다.

# 구체적지시
- `crontab/01_daily/check_oreilly`를 기준으로 Bash 스크립트를 수정한다.
- 웹페이지를 `curl -fsSL`로 가져온 뒤, 책 리스트만 포함하는 HTML 구간을 잘라낸다.
- 잘라낸 구간에서 불필요한 공백과 줄바꿈을 정규화한 다음 `sha256sum`으로 해시를 계산한다.
- 해시 비교 기준 파일은 cron 사용 계정이 쓸 수 있는 경로에 둔다.
- 이전 해시와 다를 때만 GitHub Actions를 실행하고, 성공한 경우에만 새 해시를 저장한다.
- 이 동작은 Python 코드나 Python 의존성 없이 Bash만으로 동작해야 한다.
